Swift Localized String

发表于 2018-09-10 | 分类于 IOS |

前言

iOS国际化,分为两种：

  • 1.随系统语言切换
  • 2.随用户选择切换

NSLocalizedString

  • key：字符集标识
  • tableName：strings文件的文件名，默认就是我们设置的Localizable
  • bundle：默认是mainBundle
  • comment：对key的注解，一般传空字符串

随系统语言切换

a)project -> info -> Localizations 点击加号新增，默认是英文，增加需要的语言
b)工程中新建Strings File文件，命名为Localizable（如果是自动跟随系统的话，必须这个名字
c)选中新建的strings文件，点击右边栏的Localization选项,勾选国际化的语言，此时strings文件将会变成可展开的文件，其中包含不同语言的strings文件
d)在strings文件中添加字符集（等号之前是key，之后是value），如下

d)storyboard同理，点击右边栏的Localization选项，即可实现国际化
e)InfoPlist的国际化，新建一个名为InfoPlist的strings文件，然后在添加不同语言版本的字符表
